main
VARIABLE
    jeu : Jeu
    ok : bool
    cmd : char
DEBUT
    // Charger la configuration générale
    ok ← charger_configuration("config.txt", jeu)

    si !ok alors
        écrire"Erreur : config.txt introuvable ou invalide"
    fin si

    // Charger carte 
    ok ← charger_carte("level.txt", jeu)
    si !ok alors
        écrire "erreur : level.txt introuvable ou invalide"
    fin si

    // Initialisation du jeu (joueur + monstres + brouillard)
    initialiser_jeu(jeu)

    // Boucle principale
    tant que !jeu.etat_termine faire
        afficher_jeu(jeu)
        saisir_commande(cmd)

        traiter_commande(cmd, jeu)
        mettre_a_jour_monstres(jeu)
        mettre_a_jour_visibilite(jeu)
        verifier_conditions_victoire_defaite(jeu)
    fin tant que

    // Fin de jeu
    afficher_game_over(jeu)
FIN


fonction charger_configuration(nom_fichier_config: chaine, jeu: Jeu): bool role : il lit le fichier de configuration et passe ses données dans les données traitable dans les enregistrements de configurations 
fonction charger_carte(nom_fichier_carte: chaine, jeu: Jeu): bool role : il lit le fichier de la carte et va initialisé les tableaux de tableaux représentant la map 
procédure initialiser_jeu(m→ jeu: Jeu) role : il va permettre d'initialis les donnée sde table_config symbole et config_symbole
procédure afficher_jeu(m→ jeu: Jeu) role : il va utiliser la librairie pour l'affichage et va gérer l'affichage de la carte utilisant les tableaux de tableaux de carte 
procédure saisir_commande(s→ cmd : char) role : va écouter l'entrée dans la console et va changer la valeur de cmd "commande" 
procédure traiter_commande(e→ cmd : char, m→ jeu : Jeu) role : va faire par exemple une attaque ou un déplacement du joueur
procédure mettre_a_jour_monstres(m→ jeu : Jeu) role : va mettre à jour l'état des monstres que se soit leur déplacement ou bien une attaque
prodédure mettre_a_jour_visibilite(m→ jeu : Jeu) role : va permettre de mettre à jour le tableau de tableau de carte.visible qui détermine ce que le joueur est en mesure de voir 
procédure verifier_conditions_victoire_defaite(m→ jeu : Jeu) role : va regarder ce qu'il y a de présent dans la liste des 
prcédure afficher_game_over(e→ jeu : Jeu) role : cette procédure a afficher une fin de jeu c'est à dire afficher vicroire/défaite et les stats de fin du joueur
